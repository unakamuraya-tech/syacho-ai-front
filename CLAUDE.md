# CLAUDE.md - syacho-ai-front

このファイルは、AIアシスタント（Claude）がこのリポジトリで作業する際のガイドラインを提供します。

## プロジェクト概要

- **プロジェクト名**: syacho-ai-front
- **種類**: 社長AI 7問ギャップ診断アプリ + 面談受付LP（フロントエンド）
- **技術スタック**: 静的 HTML / CSS / JavaScript（ES Modules）
- **依存関係**: なし（外部パッケージ不要）

## ディレクトリ構造

```
├── index.html          # 診断アプリ（全4画面を含む SPA）
├── lp.html             # 面談受付LP（5ブロック構成）
├── css/
│   ├── style.css       # 共通スタイル（モバイル最適化）
│   └── lp.css          # LP専用スタイル
├── js/
│   ├── app.js          # メインアプリケーション（ルーティング・画面制御）
│   ├── questions.js    # 診断コンテンツ（7問）
│   ├── scoring.js      # スコアリングロジック（3軸）
│   ├── results.js      # 結果テンプレート・デモコンテンツ・CTA文言
│   └── events.js       # イベントログ（匿名・計測用）
├── CLAUDE.md
└── .gitignore
```

## 画面構成

### 診断アプリ（index.html）
1. **トップ** (`#/`) - 診断の目的と所要時間提示、開始ボタン
2. **診断** (`#/quiz`) - 7問3択、進捗バー付き
3. **結果** (`#/result`) - 3軸スコア、結果テンプレ、CTA→LP、紹介転送テキスト
4. **デモ** (`#/demo`) - Before/After比較（会議テーマ）

### 面談受付LP（lp.html）
1. **ファーストビュー** - 見出し・サブ・CTAボタン
2. **こんな状態の人向け** - 3つの困りごと
3. **面談でやること** - 5ステップの流れ
4. **持ち帰れるもの** - 3つの成果物
5. **申し込み** - Googleフォームへの導線

## 開発コマンド

```bash
npx serve .

npx http-server . -p 3000
```

## 開発ワークフロー

### ブランチ戦略

- メインブランチ: `main`
- 機能ブランチ: `feature/<機能名>` の形式を推奨
- Claude作業用ブランチ: `claude/<タスク名>-<セッションID>`

### コミットメッセージ規約

- 日本語または英語で記述
- 簡潔で内容が明確なメッセージを心がける
- 例: `feat: ログイン画面を追加`, `fix: APIレスポンスのエラーハンドリングを修正`

## 設計方針

- MVPはルールベース（診断結果とデモは固定テンプレ）
- LLMは後から差し替え可能な設計
- 個人情報は収集しない（匿名集計のみ）
- 煽らない・断定しすぎない・攻撃しない表現を徹底
