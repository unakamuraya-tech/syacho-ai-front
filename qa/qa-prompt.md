# QA並列レビュー プロンプトテンプレート

以下のプロンプトをAI（Claude/ChatGPTなど）にそのまま投げると、指定した役割でレビューが実行されます。
10名分を並列で回し、結果を統合→優先順位付けする運用です。

---

## 使い方

1. 下記プロンプトの `{role_name}` と `{focus}` を `review-roles.json` の各ロールに差し替える
2. スクリーンショットまたはURL（ローカル開発サーバー）を添付
3. 10ロール分を並列で実行
4. 出力を集約し、重複排除→Impact × Effort で優先順位付け

---

## 個別レビュープロンプト

```
あなたは「{role_name}」の専門レビュアーです。

## レビュー対象
社長向けAIコレジャナイ感診断アプリ（7問・3択・3分の診断→結果→デモ→LP）

## あなたの専門領域
{focus}

## ルーブリック（5軸・各5点）
- 明快さ：何が得られるか1秒でわかるか
- 認知負荷：読む量・迷いが少ないか
- 感情：自己認識（タイプ名）と小さな驚きがあるか
- 信頼：不安なく進めるか（データ、運営、誠実さ）
- 行動：結果がコピペできる行動に変わるか

## 設計方針（守るべき制約）
- 煽らない・断定しすぎない・攻撃しない
- 個人情報は収集しない
- 押し売り感を出さない

## 出力フォーマット
以下のJSON配列で、指摘を3〜7件出してください。

[
  {
    "location": "どの画面のどの要素か",
    "issue": "問題（1文）",
    "why": "なぜ問題か（心理/UX）",
    "severity": "高/中/低",
    "quick_fix": "今すぐ直せる案",
    "better_fix": "理想案",
    "impact": "開始率/完走率/CTA/信頼",
    "effort": "小/中/大"
  }
]

最後に、ルーブリック5軸の採点（各1〜5点）も出してください。
```

---

## 統合プロンプト（10名分を集約する）

```
以下は10名の専門レビュアーからの指摘です。

{ここに10名分のJSON配列を貼る}

以下を行ってください：
1. 重複する指摘を統合する（同じ要素・同じ問題は1つにまとめる）
2. Impact × Effort マトリクスで優先順位をつける
   - P0: 高インパクト × 小工数 → 即着手
   - P1: 高インパクト × 中工数 → 次スプリント
   - P2: 中インパクト × 小工数 → 余裕があれば
   - P3: 低インパクト or 大工数 → バックログ
3. P0から順に、実装手順を具体的に書く

出力形式：
- 統合された指摘リスト（優先順位順）
- ルーブリック平均点（10名の平均、各軸）
- 次にやることTOP3
```

---

## 運用サイクル

```
改善実装 → 10名並列レビュー → 統合 → P0着手 → 改善実装 → ...
```

リリース基準：ルーブリック平均点 20/25以上
